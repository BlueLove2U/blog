Hexo 的基本使用方法

基础命令

  • hexo s 启动本地服务器 (server 简写)，用于本地预览博客内容 (默认端口 4000)。

  • hexo clean 清除缓存文件 (删除 public 和 .cache 目录)，用于解决生成异常/强制重新生成时使用。

  • hexo g 生成静态文件 (generate 简写)，用于将 Markdown 转换为 HTML/CSS/JS。

  • hexo d 部署到服务器 (deploy 简写)，用于配合部署插件 (如 Git, FTP) 使用。

组合命令

  • hexo clean && hexo s，等效于“清空缓存 → 启动服务”，用于确保预览内容是最新版本。

  • hexo g && hexo s，等效于“生成文件 → 启动服务”，用于快速测试生成结果。

  • hexo clean && hexo g && hexo d，等效于“清空缓存 → 生成文件 → 部署”，用于完整发布流程（推荐生产环境使用）。

参数扩展

  • hexo s --draft 预览草稿文章，用于查看未发布的 _drafts/ 目录文章。
