创建草稿

Hexo 有三种默认布局（Layout）：post、page 和 draft。 每个布局创建的文件会被保存到不同的路径。

布局 路径
post source/_posts
page source
draft source/_drafts

可以执行下列命令来创建一篇新文章或者新的页面。

如果不标识布局（Layout），则新创建的文章会会被保存到默认布局的路径下。

post 是默认的布局，文件自动生成在 source/_posts/<title>.md。也可以通过编辑 _config.yml 中的 default_layout 设置来更改默认布局。

个人建议：先生成草稿再发布到 source/_posts，而不是直接生成文章到 source/_posts。

因为在使用 hexo publish <title>" 发布文章时，Hexo 会自动帮助我们调整文章的 Front-matter 元数据，从而可以帮助我们在文章发布时检查文章内容的格式，而不是要等到在浏览器预览时才发现问题。

编辑 Front-matter

Front-matter 是文件开头的 YAML 或 JSON 代码块，用于配置写作设置。 以 YAML 格式书写时，Front-matter 以三个破折号结束；以JSON格式书写时，Front-matter 以三个分号结束。
